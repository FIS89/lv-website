LabelVision 20/20: Technical Notes

Updated License Manager for LabelVision 2000 version 3.1
This technical note explains how to resolve a problem with LabelVision 2000 version 3.1 multi-user editions.

If you are using LabelVision 2000 version 3.1 with a five or greater user license, you may see the following error message, even though you have not exceeded the licensed number of users:

LabelVision 2000 has detected another user of your license on the network. The application will be shut down.
If this happens, you probably need to get an updated License Manager.

To check if you need an update

  1. Using the Windows Explorer, open the LabelVision 2000 client installation folder.
  2. Right-click on the file PrtOnly.exe, then select "Properties".
  3. Click on the Version tab. Write down the Print Only version.
  4. Right-click on the file LcnsMgr.exe, then select "Properties".
  5. Click on the Version tab. Write down the License Manager version.
  6. If the Print Only version is 3.1.362.0 and the License Manager version is 3.1.362.0, then you must download an update to the License manager.

Do not download an update unless both file versions are exactly as shown in step 6!

To download the update

  1. Quit all LabelVision programs.
  2. Click here to download an updated LcnsMgr.exe.
  3. When prompted by your browser, choose save to file. Save the file on your hard drive. You may find it convenient to save it in the LabelVision server folder.
  4. Copy this updated LcnsMgr.exe to each of your LabelVision client installation folders, overwriting the existing file.
  5. Check the LcnsMgr.exe version in each client installation folder. It should be 3.1.385.0.
  6. Delete the file ticket.dat in the LabelVision server folder.
  7. If you install a new LabelVision client, be sure to check the version and copy the updated LcnsMgr.exe to the client installation folder after installing if necessary.
